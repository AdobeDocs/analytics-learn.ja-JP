# Google AnalyticsからAdobe Analyticsへの移行に関する包括的なガイド

## 1.はじめに

ツール間の移行における最大の課題の 1 つは、同等の機能をどこで見つけ、それを効率的に使用する方法を学ぶことです。 このディスカッションは、( 新しいユーザーとして、またはGoogle Analyticsからの )Adobe Analyticsへの移行を容易にする大きなガイドの一部です。 GA との詳細な比較ほとんどのユーザーがなじみのある最も可能性の高い比較ツールとしては、ユーザーが既存の知識を新しいツールセットと関連付けるのに役立つように提供されています。 習慣に代わるものはないし、これは、あなたが始めるのに役立ち、願わくば、この時間に遭遇する可能性のある不満を減らす（または、物事のスイングに入り始めた後のリフレッシャーとして）。

また、簡単な用語比較もおこなえます。

| **説明** | **Adobe Analytics** | **Google Analytics** |
|--------------------------------------------------------------------------------------------------------------------------------|---------------------|----------------------|
| ページ（またはアプリ上の画面）を表すイベント指標が表示された | ページビュー | ページビュー。 |
| Web サイトまたはアプリでの同じ期間に発生するインタラクションのグループを表す指標です | 訪問 | セッション |
| 識別されたデバイスを定義する指標（ユーザー情報を結び付けるための cookie および他の行動パターンを含む複数の条件に基づく） | ユニーク訪問者 | ユーザー |

## 2.インターフェイス

Adobe AnalyticsとGoogle Analyticsを比較する際によく見ることの 1 つは、Adobeが多く起きていることです。これは人々にとって大変な作業です。 これは事実ですが、それも事実です。信じようが信じようが信じようが、信じようが、信じようが、信じようが、信じようがなかった。強さでも弱さでもない。 Adobeは、データの視覚化に幅広いツールと柔軟性を提供し、必要なものをより自由に作成できます。

まず、「サイト内」レポートを見てみましょう。

### 2.1.サイト内レポート

#### 2.1.1 ホーム画面

Adobe AnalyticsとGoogle Analyticsは、両方とも、ユーザーがログインしたときに表示される最初の表示をカスタマイズする方法を提供します。

Adobe Analyticsでは、すべてのユーザーにログイン時に表示される事前定義済みレポートを作成することを推奨しません。 デフォルトのホームページでは、ユーザーが Workspace のランディング画面に移動し、各ユーザーが作成または共有したすべての Workspace レポートが表示されます。 また、各ユーザーは、これらのレポートのいずれかをホーム画面（選択した場合）として設定できます。

#### 2.1.2 サイト内Report Builder

分析ツールが提供するシンプルレポートに加えて、各ツールには、独自のカスタムレポートを作成するためのより強力なツールも用意されています。

##### 2.1.2.1. Adobe Analytics Workspace

これはAdobe Analyticsの強力な機能です。2017 年の導入以降、Analytics 分析の実現に役立ち、レポートセクションがすぐに廃止される主な理由です。

このツールを使用すると、ほぼ自由にレポートを作成できます。

レポートをパネルに分割し、それらのパネルに任意の数のビジュアライゼーションを含めることができます。 パネルは、日付範囲や共通セグメントフィルターなど、一般的な情報に設定できます。

パネルとパネル内のビジュアライゼーションの両方をサイズ変更したり、ドラッグして項目を並べて表示したり、積み重ねたりできます。 したがって、2 つの異なるデータスイートを並べて比較する場合は、中央に分割さ50/50、2 つのサイトを並べて表示し、比較しやすくするパネルを作成できます。

ユーザーは多数のビジュアライゼーションを使用できます。

* フリーフォームテーブル
* コホートテーブル
* フォールアウト
* フロー
* グラフ
   * 面グラフ（積み重ねおよび積み重ね解除）
   * 行
   * 散布図
   * 棒グラフ（積み重ねおよび積み重ね解除）
   * 箇条書き
   * ドーナツ
   * ヒストグラム
   * 横棒グラフ（積み重ねおよび積み重ね解除）
* マップ
* 概要ブロック
   * 変更概要
   * 概要テキスト
   * テキスト（コンテキストを提供する追加の情報を入力するフリーテキストフィールド）
* ベン

各パネルとビジュアライゼーションには、情報が何を示しているかに関するコンテキストを提供するために、タイトルを設定し、説明を適用することができます。
Adobeでは、セグメント（基本的にはデータのフィルター）が遡って適用され、フリーフォームテーブルの列に取り込んで、データを並べて比較できます。 例えば、ユーザーがサイト上でトラフィックに関して 2 つの異なるカテゴリを比較したい場合、「カテゴリ A」に対してセグメントを作成し、「カテゴリ B」に対して別のセグメントを作成できます。

フリーフォームテーブルでは、必要に応じて、複数の列とセグメントを使用し、データを希望の方法で視覚化できます。

上記の内訳を日付別に表示したくない場合は、 別のディメンションまたはセグメントをドラッグ&amp;ドロップして別の方法でデータを表示します。例えば、デバイスタイプにセグメントを使用し、モバイル/タブレットユーザーに OS で分類を追加します。

![image3](assets/ga-to-aa_4.png)

Workspace では、「標準」の分類に限定されず、クリエイティブを飛ばすことができます。 実行する必要のある比較について詳しく知る必要のあるビジュアライゼーションを構築できます。

また、レポート内にのみ存在する計算指標やセグメントを即座に作成することもできます（セグメントや計算リポジトリの洪水を防ぎます）。また、組織を他のコンテキストではあまり使用できないものと混同することなく、特定のレポートに必要な重点項目を作成できます。

この説明は、このツールの紹介にすぎません。また、より包括的なガイドを使用して作業を開始できますが、その際に、次のような包括的なレポートを作成できます。

ワークスペースのもう 1 つの強力な機能は、インタラクティブ修飾子をドロップダウン形式でレポートに適用できる点です。 これらのドロップダウンは、レポートの書き出した CSV やPDFファイルでは機能しませんが、ライブレポート内では、パネル内のすべてのビジュアライゼーションを更新し、異なる条件下で同じレポートを表示できます。 複数のドロップダウンを使用でき、オプションが相互に排他的でない限り、選択した項目は積み重ねられ、情報を明確に表示できます。

##### 2.1.2.2.Google Analytics:ダッシュボード、カスタムレポート、保存済みレポート

Googleには、インターフェイス内でレポートを作成するためのツールがいくつかありますが、レポートセクションの表示と制限は同じです。

Google Analyticsに詳しい方はこう言うかもしれません「ちょっと待ってGoogle Data Studio はAdobeの Workspace と同等のものではないか？」 正しいと思いますが、Data Studio は Analytics ツールの一部ではなく、様々なデータソースへの接続を可能にするので、このツールについては、このディスカッション（特に 2.2.3 節）の「拡張レポートアクセス」の節で説明します。

Googleのダッシュボードおよびカスタムレポートを使用すると、複数のビジュアライゼーションを 1 つのレポートにまとめることができますが、Workspace とは異なり、単純な相関と、どのデータをどの列に配置できるかにロックされています。

カスタムレポートで最も大きな課題の 1 つは、フィルターを作成する際に、そのフィルターがレポートのすべてのタブに適用される点です。同じレポート内で 2 つの異なるフィルターを比較する方法はありません。

サーフェス比較の場合は、この操作が実行されます。 これらはすべて、Adobeのレガシーダッシュボード、カスタムレポートおよびブックマークに似ています。 ニーズをサポートするために提供される、レポートスイート内の基本ツール。

#### 2.1.3 報告

GoogleとAdobeの両方に、ディメンションに基づく、ビルドテーブルおよび基本的なタイムライングラフで構成されるナビゲーション可能なレポートがあります。

##### 2.1.3.1.Adobe Analyticsレポート

Adobe Analyticsには「レポート」セクションもありますが、大部分は廃止され、Analysis Workspaceに置き換えられています（実際には、Workspace 以降、このインターフェイスの提供終了が発表されています）。 [セクション 2.1.2.1] は、より強力なツールです )。この場合、これらのテーブルのほとんどを簡単に作成および変更できます。 Adobeのセクションは、より詳細に分類されており、これは非常に困難です。

![image3](assets/ga-to-aa_6.png)

上記のほとんどが Workspaces を通じてアクセスできるので、これらの節の概要とGoogle Analyticsとの関係を簡単に説明し、まだ関連性の高いレポートをここで強調します。

サイト指標は期待されるもので、標準指標（ページビュー数、個別訪問者数、訪問回数、および設定したカスタムイベント）が含まれます。 これは行動レポート GA に似ていますが、オーディエンスに表示される内容の一部も含まれます (Adobeは指標タイプを分割しないので )。

ここでは、「ボット」レポートも見つかります。ボットからのトラフィックはすべての標準レポートから除外されますが、2 つのレポートがあり、何が起こっていて、どのボットがサイトに来ているかを把握できます。 これは、サイトに頻繁にヒットする既知のスパムボットを除外するカスタムボットルールを設定する場合に特に便利です。 主なレポートが水浸しになってもトラフィックが発生しない場合に、これらのボットが何を実行しているかに関するインサイトを得ることができます。 現在、ボットレポートは Workspace からは使用できません（ただし、近日中に新しいレポート機能が追加され、ユーザーがこの情報をそこに取得できるようになります）。

サイトコンテンツは、次のAdobe標準ディメンションのグループです。ページ名、サイトセクション（チャネル）、階層（Web サイト内の組織のカスタムドリルダウンレポートを作成する方法）、サーバー（サイトに複数のサブドメインがある場合、または複数のサイトを 1 つのトラッキングスイートにタグ付けする場合に特に役立ちます）など。 これらはすべて、Workspace で使用できます。

モバイルは、デバイス、デバイスタイプなど、モバイルデバイス固有のデータをグループ化したものです。 これらはすべて、Workspace で使用できます。

パスは、「Workspace ではまだ利用できません」項目の 1 つです。Workspace にはフロー図がありますが、単一のページ/値の入出力フローのみが表示されます。パスは、Web サイトで使用される最も一般的なパスを表示できます。 デフォルトでは、ページはユーザーに代わって設定される最初のパスレポートですが、カスタム prop（「ページタイプ」値を追跡する場合など）に対してこれをオンにすると、ページタイプ内のパスを確認できます。 私が個人的に気に入っているもう一つの点は、情報を簡単に表示する方法です…ワークスペースのフロー図（どの程度を見ようとしているかによって異なります）は圧倒される可能性があります。 両方を試してみることをお勧めします…それぞれが、何を達成しようとしているかに応じて、使用と価値を持っています。 フローでは任意のディメンションを使用できますが、パスは管理パネルの Prop で設定する必要があります。

トラフィックソース、キャンペーン、マーケティングチャネルの各レポートは、すべてGoogleの獲得レポートに似ています。 トラフィックソースは実際のリファラーに焦点を当て、キャンペーンはキャンペーンコードに焦点を当て、マーケティングチャネルはキャンペーンコードにも焦点を当てますが、情報の処理方法に基づいて追加のロジックも適用します。 Adobeはルールを設定する方法をより自由に使えるのですGoogleは多くの事を行いますだから少し考え方が変わります また、デフォルトでは、Googleのキャンペーンコードのアトリビューションは 6 か月で、Adobeのアトリビューションはデフォルトで 1 週間に設定されます。 これは管理設定で変更できますが、Workspace では、実際には任意のディメンションの上にカスタムアトリビューションを適用し、より多くの「その場で」柔軟に適用できます。

訪問者保持率レポートと訪問者プロファイルレポートは、訪問者のオーディエンスレポートに似ていますGoogle Analytics。 リテンションは、再来訪頻度に重点を置いていますが、訪問者プロファイルは、ユーザーの地域と技術に重点を置いています。

カスタムコンバージョンおよびカスタムトラフィックは両方ともカスタムディメンションレポートで、コンバージョンは eVar です ( コンバージョンは、値（ヒット、訪問、月、年など）にカスタム有効期限を設定できます。この値は、上書きされない限り、指定された時間保持されます )。 トラフィック変数は prop ですが、パスレポートやリスト項目用に設定することもできます（選択した区切り文字に基づいて複数の値を分割します）。

メディアは、特別なメディアトラッキングを設定したビデオやオーディオファイルなどの用途に使用します。

カスタムレポートは、ユーザーがレポートインターフェイス内で作成した列と分類をカスタマイズし、カスタムレポートとして保存できるセクションです。 ただし、前述のように、Workspace では非常に強力な分類と相関関係が可能なので、カスタマイズされたものはすべてそこに作成する必要があります。 これは、Workspace が存在する前の良い解決策でした。

「ブックマーク」セクションは、カスタムレポートに似ています。カスタムレポートでは、頻繁に使用するレポートをレポートインターフェイス内でブックマークして、見つけやすくすることができます。

ダッシュボードは、人々がデータのレポートレットを 1 つのビジュアライゼーションに組み合わせることができる従来の製品でした。 ただし、Workspace の機能（セクション 2.1.2.1）の操作は非常に簡単なので、この機能が廃止される前に再構築する必要がある従来のレポートへのアクセスポイントとしてのみ存在します。

Targets は、特定の期間内にターゲットに基づくレポートを作成できる特別なレポート領域で、チームはキャンペーンなどの状況を監視し、トラフィックターゲットに到達する軌道に乗っているかどうかを確認できます。

ここにあるすべてのレポートは、複数の指標列とディメンション分類に使用できます。 しかし、ビジュアライゼーションの単純さと、相関を持つ可能性のある要素の背後にあるロジックの一部は、時々不満になる可能性があります。

##### 2.1.3.2Google Analyticsレポート

Google Analyticsは、これらのレポートを次のセクションに分割します。リアルタイム、オーディエンス、獲得、行動および会話 (GA3) とライフサイクル（サブセクションを使用）:獲得、エンゲージメント、収益化、定着 ) およびユーザー（以下のサブセクションを使用）デモグラフィックとテクニカル )。

![image3](assets/ga-to-aa_7.png)

これらのビジュアライゼーションに小さな調整を加えたり、二次ディメンションの分類を追加したり、ビジュアライゼーションを変更したり、データに対するフィルターを作成したりできます。 カスタマイズ内容を保存済みレポートとして保存できます。

これらのツールを使用すると、データに関する迅速で簡単なインサイトを得ることができます。 ただし、同じテーブル内のページのユーザーとページビューを比較することはできません。また、追加のデータを表示するために複数のディメンションを追加することもできません。

これらは迅速な分析データに適していますが、深く掘り下げる必要がある場合は、制限を受けます。

### 2.2.レポートへのアクセスの拡張

「サイト内レポート」に加えて、ほとんどのツールには拡張機能が用意されており、ツール以外の場所で分析を実行し、さらにカスタマイズされたものを構築できます。

#### 2.2.1. Adobe AnalyticsReport Builder(Microsoft Excel Extension)

Workspace は優れたツールですが、場合によっては、データをカスタマイズしたスプレッドシートに取り込み、複数のデータソースを結び付ける必要があります。 ここがReport Builderの役目です

Report Builderは、Adobe Analyticsデータへの接続を作成して、Excel 内で操作できる表形式のデータを取り込むことができる、Microsoft Excel のプラグインです。 一般に、これを効率的に使用するには、データを一部の生データタブに取り込み、Excel セル参照を使用してこれらのタブのデータを単一の統合レポートに取り込み、グラフとビジュアライゼーションを作成します。

>[!NOTE]
>
>Report Builderは、このプラグインにアクセスするためにユーザーに適用する必要がある特別な権限を持っています。 これは、ツールの正しい使用方法を学んだユーザーにのみ付与される必要があります。

#### 2.2.2. Adobe Analytics API 接続

Adobe Analyticsを Excel 以外のもので消化する必要がある場合でも、処理されたデータ（ボットルールの除外を含む）の利点が必要な場合は、Adobeの API を使用して直接データを取り出し、スクリプトで処理するか、別のシステムで使用するためにデータベースに追加できます。

API は引き続き、プルリクエストで指定された分類とセグメントを適用する相関データを取り込むことに注意してください。

Adobeの Workspace（セクション 2.1.2.1）は、実際に API を使用してすべてのレポートを作成します。Workspace でデバッグモードを有効にすると、使用された正確な API 呼び出しが表示されます。 これは、Workspace を使用して取り込むデータを作成および検証し、それらの API 呼び出しを使用して独自の処理でデータを取り出して API 呼び出しを簡単に作成する方法です。


#### 2.2.3.Google AnalyticsData Studio

読み進めた方は、上記から、Data Studio をAdobeの Workspace と同等のものとして挙げたことを既にご存知です。 Data Studio を使用すると、Google Analyticsデータを取り込むことができますが、他のソースからのデータも取り込むことができます。 これは、分析データを他の収集データと統合する場合に便利です。しかし、Google Analyticsに関しては、Google Analyticsに存在するのと同じような視覚化の制限が見つかりました。 行と列の形成方法は、実行できる処理に非常に制限があります。

これはまだ強力なツールであり、私は人々にそれを使用することを絶対に避けようとはしませんが、私の個人的な経験は、Workspace を長い間使用してきた私は個人的には、厳しい行動を非常に限定的に見つけることです。


#### 2.2.4. Google Spreadsheet Extension

Google Analyticsからデータを拡張して取り込む必要がある場合、私が選択する個人用ツールはGoogle Spreadsheet Extension です。 確かに、GA テーブルに複数のReport Builderを接続する必要がありますが、Adobeの接続と同様に、生データからセルを参照し、必要なレポートを作成して、Googleスプレッドシートのグラフ化機能を使用して視覚化できます。


## 3.生データのエクスポート

AdobeとGoogleの両方が、このような方法で情報を取り込む機能を提供しています。

### 3.1.Adobeデータフィード

2.2.2 節では、Adobe Analytics API が「処理済みのデータ」から取り出したことを述べました。 生データフィードは、引き続き管理パネルで設定された「処理ルール」で処理されるデータを取り込みます（生データの取り込み時までにこれらのルールがすべて完了するように、生データを遅延させます）。

つまり、すべてのボットの除外、内部 IP フィルター適用済みデータなどが生データフィードに含まれます。 このデータを識別するフラグがあるので、データレイクを構築する場合は、エンジニアリングチームがそのデータを処理するロジックを作成できます。

生データフィードは、データのすべての列を送信するようにカスタマイズすることも、より焦点を絞ったフィードが必要な場合は特定の列のみを送信することもできます。

フィードは、FTP、SFTP、S3 などに直接送信できます。


### 3.2. Googleビッグクエリ

残念ながら、これはGoogleツールの 1 つです。これは、私が使用した経験はありませんが、理論上はAdobeのデータフィードと似ており、エンジニアリングチームがGoogle Analyticsアカウントから生データにアクセスできるようにする必要があります。

ただし、エンジニアは SQL クエリを使用してデータにアクセスできるので、対象となる生データを取り込むか、生データの列をすべて取り込んでデータレイクに取り込みたい場合は取り込むことができます。
