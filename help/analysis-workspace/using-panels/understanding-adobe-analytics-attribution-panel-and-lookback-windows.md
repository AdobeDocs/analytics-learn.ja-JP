## **アトリビューションモデル**

この **アトリビューションパネル**&#x200B;では、いくつか異なる観察を始めるかもしれません。  例えば、 **アトリビューションモデル** 我々が *コンバージョン* ( 例： ❶ **成功指標**) は、 *ヒット数* 任意のグループ内。

簡単に言えば、 **10 人** 押す **大きな赤いボタン** 戸を通る **アトリビューションモデル** どちらが **10 人** 「クレジット」を割り当てたい、あるいはより良い言葉を割り当てたい。 *多い* 「クレジット」を割り当てます。

![ボタン](assets/button.png)

これを念頭に置いて、以下に、 **アトリビューションモデル** それらに影響を及ぼす可能性がある **10 人**:

- **ファーストタッチ**:このモデルは、 **100%のクレジット** から *first* ドアを通り抜けた人物  マーケターは、このアプローチを ***ソーシャルメディア*** または ***表示***;ただし、オンサイト製品のレコメンデーション効果に頻繁に使用する優れた戦術でもあります。
- **ラストタッチ**:この戦術もまた、聞こえるとおりに機能しますが、代わりに次のようになります。 **100%のクレジット** 扉を抜けた最後の男に  このモデルは、通常、 ***自然（自然）検索*** その他 *短期間* マーケティングサイクルキャンペーン。
- **線形**:このモデルは、ドアを通り抜けた全ての個人に対し、同じクレジットを配分します。

  >[!CAUTION]
  >
  >ただし、この戦術を適用する際には、実行が長くなり、ヒットするオーディエンスが多くなると考え、結果を非常に薄くする可能性があるので、注意が必要です。

- **U 字形**:このアプローチは、 **40%** ～に対する信用の *第一人者* 戸口に広がる **20%** ～全体の信用の *その間の全員*&#x200B;を **40%** から **最後の 1 つ** から このモデルは、 **長いコンバージョン/販売サイクル** 次を含む *複数のタッチポイント* 途中で  この場合、主に ***first*** および ***last*** 顧客のコンバージョンに貢献したマーケティング戦術。
- **J**-**シェイプ** および **逆 J 形**:
   - 考える **U 字形**&#x200B;が、代わりに、このモデルが **60%** ～に対する信用 *姓* 扉を通り抜け **20%** から *first*、 *除算* 残りの **20%** 横 *他の全員* 真ん中に  **逆 J 形** 正反対のことをする

     ここでの目標は、 *開始* または *終了* キャンペーンのしかし、途中で「小さな男」を認識しながら、その反対側の投稿項目に一定額のクレジットを割り当てたいと思うのです。

- **タイムディケイ**:これを分かち合わなかったら寂しいと思います このモデルは文字通り、指数的に減少する半減期 — 時間の経過と共に！  この場合、 *デフォルト* このモデルの半減期のパラメーターは次のとおりです。 **7 日**.  その仕組みは、次に適用することです *重み付け* 各 **マーケティングチャネル**, *時間に基づいて* それは *初期タッチポイント* 顧客がコンバージョンしたとき。

  **タイムディケイ** および **U 字型アトリビューションモデル** はどちらも、一般的に、長い名前のキャンペーンの測定に使用されますが、ご覧のように、最終的な目標はどのようにして異なる目標を持っています *重み付けする* 結果の値。

## **ルックバックウィンドウ**

次のレベルに向かって考える時が来た。  ここでは、文字通り時間旅行要素を分析に追加します。ここでも、基本について説明します。

***Adobe*** 定義 **ルックバックウィンドウ** を「コンバージョンがタッチポイントを含むように振り返る必要がある時間。 最初のインタラクションにより多くのクレジットを与えるアトリビューションモデルでは、異なるルックバックウィンドウを表示する際に、より大きな違いを確認できます。」


つまり、 **ルックバックウィンドウ** 期間を決定する *コンバージョン* 考慮され、 *context* を属性分析に追加します。 ***Adobe Analytics*** は、次の 3 つのタイプを提供します。 **ルックバックウィンドウ**:

- **訪問のルックバックウィンドウ：** 最初の ***訪問*** コンバージョンが発生した場合に、コンバージョンにつながる即時のインタラクションに関するインサイトを提供します。

  通常、これは最も短いことを覚えておいてください **ルックバックウィンドウ** を使用します。
- **訪問者のルックバックウィンドウ：** 全てを見る ***訪問回数*** 選択した月の最初までをバックアップ **日付範囲**&#x200B;を使用すると、顧客のインタラクションに関するより広い視野が得られ、経時的なパターンの識別に役立ちます。
- **カスタムルックバックウィンドウ：** を展開できます。 **属性ウィンドウ** 報告を超えて **日付範囲** 最大 *最大* / **90 日**.  以下を提供します。 *柔軟性* 発生したタッチポイントのキャプチャ *外部* 選択した **日付範囲**：包括的な分析を確実におこないます。

特定の **ルックバックウィンドウ**&#x200B;その後、アナリストは、特定の期間内での 1 つ以上のタッチポイントの影響を調べ、異なる期間がアトリビューション結果に及ぼす影響についてのより詳細なインサイトを得ることができます。

## **まとめ**

これはアナリストにとって何を意味するのでしょうか？

この **アトリビューションパネル** および **ルックバックウィンドウ** 日常的で表面レベルのデータを超えて見渡し、カスタマージャーニーを深く掘り下げる力を私たちに与えてください。 どのタッチポイントが～に最も大きな影響を与えたかを把握することで *コンバージョン*&#x200B;では、十分な情報に基づいたマーケティング戦略の決定を下し、より効果的にリソースを割り当てることができます。

次に **アトリビューションモデル** および **ルックバックウィンドウ** を選択した場合でも、を使用してフィルタリングすることで、データをさらに操作できま❺す **セグメント** またはこの時点で必要なその他のコンポーネント  さらに、パネルがレンダリングされると、従来の Workspace のすべての機能を自由に使用できます。
